# Active specs

Put active design specs here while they are shaping implementation work.

Spec lifecycle:

1. Create a spec when a change needs design alignment before task breakdown.
2. Link the follow-on plan from the spec once execution starts.
3. When implemented, move durable decisions into ADRs, inline READMEs, AGENTS, or
   source comments.
4. Delete or archive the spec when it is completed, superseded, or abandoned.

Completed specs should not remain in this folder.
