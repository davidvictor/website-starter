# `src/components/ui/` — shadcn primitives

> **Out of bounds for direct edit.** These files are regenerated by the shadcn CLI; hand-edits get clobbered on upgrade.

## What's here

shadcn/ui primitives (Base UI variant): `Button`, `Card`, `Dialog`, `Sheet`, `Tabs`, `Tooltip`, `Toast`, etc. Every shadcn component is pre-installed so block variants and pages can compose without `pnpm dlx shadcn add` runs.

## Why off-bounds

These files are tracked in `biome.json` `experimentalScannerIgnores`. Biome doesn't lint them; humans shouldn't either. When shadcn ships an upgrade (new API, new accessibility fix, new design tweak), regenerating these files is a one-line CLI run. Any hand-edits you made disappear in that regeneration.

## How to customize

**Don't edit these files.** Instead, wrap them.

Example: you want a "marketing CTA" button that's always primary, always large, always has an icon. Wrong path:

```tsx
// ❌ Edit src/components/ui/button.tsx to add an iconLeft prop
```

Right path:

```tsx
// ✅ Create src/components/cta/cta-button.tsx
import { Button, type ButtonProps } from "@/components/ui/button"
import { ArrowRight } from "lucide-react"

type Props = Omit<ButtonProps, "size" | "variant"> & {
  iconRight?: boolean
}

export function CtaButton({ iconRight = true, children, ...rest }: Props) {
  return (
    <Button size="lg" variant="default" {...rest}>
      {children}
      {iconRight && <ArrowRight className="ml-2 h-4 w-4" />}
    </Button>
  )
}
```

The wrapper lives in a feature folder, depends on the primitive, and survives shadcn upgrades.

## What about theme tokens?

Sitewide visual changes (color, radius, spacing scale) flow through `src/themes/` and `src/app/globals.css`. The primitives in this folder consume CSS variables (`var(--primary)`, `var(--radius)`, etc.) — change the variable, change every primitive at once. **Never** hardcode a color or radius inside a primitive to "match a design."

## Upgrading shadcn

```bash
pnpm dlx shadcn@latest add button --overwrite
```

Run on a clean branch, review the diff carefully (some primitives have had small patches applied in the past — see the "Limitations" section in the top-level `README.md`), commit as a `chore(deps): upgrade shadcn primitives`.
